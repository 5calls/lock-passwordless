The UI Options will help you customize the look and feel for your Passwordless Lock. Take a look at the full spec of options.

Url for an image that will be placed in the Lock's header. Defaults to Auth0's logo.
Defines the primary color of the Lock, all colors used in the widget will be calculated from it. Defaults to '#ea5323'.
The id of the html element where the Lock will be rendered. This makes the Lock appear inline instead of in a modal window.
ISO country code of the country that will be selected by default when entering a phone number. Defaults to 'US'.
Allows to customize every piece of text displayed in the Lock. Defaults to '{}'. See the full Dict Specification for more details

The Authentication Options will allow you different ways of triggering the sign in and saving states for your user. Take a look at the full spec of options.

Specifies extra parameters that will be sent when starting a login. Defaults to '{}'.
The url Auth0 will redirect back after authentication. When provided it will enable redirect mode. Defaults to the empty string "" (no callback URL).
Should be set to 'token' for Single Page Applications, and 'code' otherwise. Defaults to 'code' when callbackURL is provided, and to 'token' otherwise.
